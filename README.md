# Автопортал «Ангар» – система автоматизации грузовых перевозок

Веб-приложение для транспортной компании **ООО «Ангар»**, позволяющее централизованно учитывать рейсы, пробег автомобилей, автоматически рассчитывать зарплату водителей, вести учёт затрат на ТС и формировать аналитические отчёты.

## Основные возможности

- **Аутентификация и разграничение ролей**  
  - Администратор, диспетчер, водитель. Каждая роль имеет свой набор прав.
- **Управление рейсами**  
  - Создание, редактирование, удаление (только запланированных).  
  - Завершение рейса с автоматическим расчётом пробега и зарплаты водителя по тарифу за км.  
  - Фильтрация рейсов по дате, водителю, автомобилю, статусу.  
- **Управление автопарком**  
  - Добавление, редактирование, удаление автомобилей (удаление только для администратора).  
  - Контроль текущего пробега и статуса (доступен, в рейсе, ремонт).  
- **Управление водителями**  
  - Создание водителей с привязкой к пользователю (пароль, email).  
  - Индивидуальный тариф за километр.  
  - Редактирование профиля, смена пароля.  
- **Учёт затрат**  
  - Фиксация расходов по автомобилям (категория: топливо, ремонт, обслуживание, прочее).  
  - Фильтрация по дате, автомобилю, категории.  
- **Аналитика и отчёты**  
  - Сводная статистика (количество рейсов, общий пробег, зарплата, затраты).  
  - Графики (пробег по водителям, затраты по автомобилям).  
  - Таблицы с детализацией по водителям и автомобилям.  
- **Современный UI/UX**  
  - Адаптивный дизайн (TailwindCSS).  
  - Иконки Font Awesome.  
  - Всплывающие уведомления с автоматическим скрытием.  
  - Модальное окно для завершения рейса с динамическим расчётом зарплаты.

## Требования к окружению

- **Операционная система:** Windows (Laragon), Linux, macOS (с аналогами).
- **Веб-сервер:** Apache (рекомендуется) или Nginx.
- **PHP:** версия 8.1 или новее (модули: `pdo_mysql`, `session`, `json`, `random`).
- **MySQL:** версия 8.0+.
- **Composer:** для автозагрузки классов.
- **Git:** для клонирования репозитория (опционально).

## Установка и запуск (Laragon 6)

### 1. Установка Laragon

Скачайте и установите [Laragon](https://laragon.org/download/) (версия 6.0 или выше).  
Убедитесь, что в состав входят Apache, MySQL 8+, PHP 8.1+.

### 2. Клонирование или распаковка проекта

Поместите код проекта в папку `C:\laragon\www\angar-portal`.

```bash
cd C:\laragon\www
git clone https://github.com/your-repo/angar-portal.git
# или просто скопируйте файлы вручную
```

### 3. Настройка виртуального хоста

Laragon автоматически создаст виртуальный хост по имени папки.
Откройте терминал Laragon (правой кнопкой мыши → Terminal) и выполните:

```bash
cd angar-portal
composer install
```

Если Composer не установлен глобально, скачайте composer.phar и выполните php composer.phar install.

### 4. Создание базы данных

Через меню Laragon: MySQL → CLI или phpMyAdmin (http://localhost/phpmyadmin).
Выполните:

```bash
CREATE DATABASE IF NOT EXISTS angar_portal CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

Или

Откройте терминал Laragon (правой кнопкой мыши → Terminal) и выполните:

```bash
cd angar-portal
php migrate.php
```

### 5. Конфигурация окружения

Скопируйте пример файла .env (если его нет – создайте в корне проекта):

```bash
DB_HOST=localhost
DB_NAME=angar_portal
DB_USER=root
DB_PASS=
```

Для Laragon пароль по умолчанию пустой. При необходимости измените.

### 6. Выполнение миграций

В корне проекта есть скрипт migrate.php. Запустите его:

```bash
php migrate.php
```

Он выполнит по порядку:
- migrations/001_create_tables.sql – создание таблиц.
- migrations/002_seed_data.sql – добавление ролей и администратора.
- migrations/003_seed_demo_data.sql – заполнение демо-данными (водители, авто, рейсы, затраты).

Вы увидите:

```text
База данных `angar_portal` создана или уже существует.
Выполнен: 001_create_tables.sql
Выполнен: 002_seed_data.sql
Выполнен: 003_seed_data.sql
Миграция завершена успешно.
```

### 7. Проверка прав доступа к папкам

Laragon обычно не требует дополнительных прав, но убедитесь, что веб-сервер имеет доступ к папкам app/, public/, config/, migrations/.

### 8. Запуск приложения

Откройте браузер и перейдите по адресу:
http://angar-portal.test/

Если порт 80 занят, Laragon может использовать другой (например, http://angar-portal.test:8080). Узнать можно через меню Laragon → Apache → Ports.

# Основные маршруты (Routes)

Ниже перечислены все доступные маршруты веб-портала «Ангар».  
Метод `GET` – для отображения страниц и получения данных, `POST` – для отправки форм (изменение данных).

## Аутентификация

| Метод | URL         | Описание                         | Доступ      |
|-------|-------------|----------------------------------|-------------|
| GET   | `/login`    | Форма входа                      | Гость       |
| POST  | `/login`    | Проверка учётных данных и вход   | Гость       |
| GET   | `/logout`   | Выход из системы                 | Любой       |

## Главная

| Метод | URL | Описание                   | Доступ          |
|-------|-----|----------------------------|-----------------|
| GET   | `/` | Дашборд (статистика, последние рейсы) | Авторизованный |

## Рейсы (`/trips`)

| Метод | URL                       | Описание                             | Доступ (роль)                |
|-------|---------------------------|--------------------------------------|------------------------------|
| GET   | `/trips`                  | Список рейсов с фильтрацией          | admin, dispatcher, driver*   |
| GET   | `/trips/create`           | Форма создания рейса                 | admin, dispatcher            |
| POST  | `/trips`                  | Сохранение нового рейса              | admin, dispatcher            |
| GET   | `/trips/{id}/edit`        | Форма редактирования рейса           | admin, dispatcher            |
| POST  | `/trips/{id}`             | Обновление рейса                     | admin, dispatcher            |
| POST  | `/trips/{id}/complete`    | Завершение рейса (расчёт пробега и зарплаты) | admin, dispatcher |
| POST  | `/trips/{id}/delete`      | Удаление рейса (только запланированного) | admin, dispatcher |

> *Водитель видит только свои рейсы (фильтрация на уровне контроллера).

## Автопарк (`/vehicles`)

|-------|---------------------------|--------------------------------------|------------------------------|
| GET   | `/vehicles`               | Список автомобилей                   | admin, dispatcher, driver    |
| GET   | `/vehicles/create`        | Форма добавления автомобиля          | admin, dispatcher            |
| POST  | `/vehicles`               | Сохранение автомобиля                | admin, dispatcher            |
| GET   | `/vehicles/{id}/edit`     | Форма редактирования                 | admin, dispatcher            |
| POST  | `/vehicles/{id}`          | Обновление автомобиля                | admin, dispatcher            |
| POST  | `/vehicles/{id}/delete`   | Удаление автомобиля                  | admin                        |

## Водители (`/drivers`)

| GET   | `/reports`                | Страница аналитики (графики, таблицы) | admin, dispatcher            |

## Вспомогательные API

| Метод | URL                                 | Описание                                   | Доступ          |
|-------|-------------------------------------|--------------------------------------------|-----------------|
| GET   | `/api/trip-driver-tariff/{tripId}`  | Получение тарифа водителя для рейса        | admin, dispatcher (используется в модалке) |

## Параметры маршрутов

- `{id}` – числовой идентификатор сущности (рейса, автомобиля, водителя, затраты).
- `{tripId}` – числовой идентификатор рейса.

## Примечания по безопасности

- Все `POST`-запросы требуют наличия валидного CSRF-токена в поле `csrf_token`.
- Доступ к маршрутам ограничен на уровне контроллера с помощью методов `Auth::isAdmin()`, `Auth::isDispatcher()`, `Auth::isDriver()`.
- Водитель не имеет доступа к созданию/редактированию рейсов, автопарка, водителей, затрат – только чтение своих рейсов.

## Примеры URL

- Список рейсов за май 2025:  
  `GET /trips?date_from=2025-05-01&date_to=2025-05-31`
- Редактирование автомобиля с ID 3:  
  `GET /vehicles/3/edit`
- Завершение рейса с ID 15:  
  `POST /trips/15/complete` (с параметром `end_mileage` в теле запроса)
